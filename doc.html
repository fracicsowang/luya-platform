1. 概述 · Overview

Luya 是一套室内微菜种植系统：智能种植机（中国代工）+ 预播种子盘 / 营养液（美国生产）+ App + 云。

核心定位：Luya Platform 不是传统 ERP，而是一个 IoT 产品运营平台（设备云 + 制造门户 + 轻量客户/订阅）。 Not an ERP — an IoT product-operations platform (device cloud + manufacturing + light customer/subscription ops).

销售在外部后台运营：Shopify（独立站）+ Amazon 各自后台管订单/收款/退货；Luya 只只读同步订单进来对接设备与客户，不重建销售系统。 Sales run in Shopify/Amazon backends; Luya pulls orders read-only to link devices/customers.

2. 两条身份线 · Product identity vs Device identity

「商品身份」是品类级（一款商品），「设备身份」是单台级（一台机器）。一个 SKU 下有成千上万台不同 SN。

商品身份 Product SKULuya 内部码—印盒上→ UPCGS1 全球条码—上架 Amazon→ ASINAmazon 编号
设备身份 Device SN这一台序列号—出厂生成→ UUID云端真身
标识谁生成何时 / 用途
SKULuya 自定义立项建商品时；订单/库存/工单内部用，可自由定义
UPC (GTIN)GS1 申请前缀后自分配上架前；每个可售 SKU 一个，印在零售彩盒外。GS1 官方申请基本当天拿到；勿买转售码（Amazon 校验品牌归属会拒登）
ASINAmazon 自动生成上架 Amazon listing 时；仅 Amazon 体系用，不可自定义

连接点：订单行用 SKU 下单 → 履约 / 激活落到具体 SN。SKU=哪款，SN=哪一台。

3. 设备 ID 谱系 · Device ID lineage

每个设备相关 ID：何时生成、含义、可见性。这是「Luya 掌控设备身份」的核心——哪怕工厂没有 MES。

SN · 公开身份

印机身/箱外+二维码，到处查；公开，不含密钥，且连续可枚举。

认领权 · 蓝牙近场

无激活码卡：在机器旁用蓝牙近场读到 SN，激活在连 Wi-Fi 这一步完成 = 证明物理占有。

UUID · 云端真身

永不变、不外露；证书/MQTT 认它，换标签也改不了身份。

ID生成环节含义可见性
work_order_number开单（运营）批次号，质量追溯到生产批内部
serial_number (SN)工位 A 生成（系统）人读主键 LYX-YYMM-######，全链路查它公开
device_uuid工位 A 生成永不变的云端真身，证书/MQTT 绑它内部
device_secret工位 A 生成设备↔云通信凭证（TLS/证书）；不印卡、不给客户内部
qr_token工位 A 生成二维码里随 SN 一起编入；防伪/防枚举，绑定时校验私密
部件 SN：PCB/相机/水泵/LED/电源工位 A 绑定（工人扫枪）部件级追溯，RMA/批次召回内部
wifi_mac / bt_mac工位 A 绑定网络身份，配网/识别半公开
mcu_id工位 A 绑定主控芯片唯一号，防伪/绑定校验内部
firmware_version工位 A 烧录 / 后续 OTA当前固件，可 OTA 更新内部
owner (email)客户绑定（扫码+邮箱）设备↔账户归属；解绑即恢复出厂、可转让内部
grow / recipe / tray_id首次种植（扫种子盘）把这台的一次生长串起来内部

主线：工单 → 生成 SN/UUID → 部件 SN 挂到 SN 名下 → 印标签发货（外部只有 SN+二维码）→ 客户邮箱登录 + 蓝牙近场读 SN，连 Wi-Fi 这一步完成激活 → 写 owner → 云端用 UUID 作真身 → 种植/客服都用 SN 反查全部上下文。

4. 设备制造（中国工厂 · 多工位扫码）· Device manufacturing

即使工厂没有 MES，每台设备也必须先在 Luya 制造门户「出生」。下单 100 套 = 一张工单，批量预生成身份。

第 1 步：批量预生成（主管）

系统一次性发放 100 个 SN / UUID / device_secret / qr_token，并提前打印整卷二维码标签分发到工位。

三个独立工位（每个 = 一名工人的屏）

工位 A · 装配扫码 → 绑定部件 SN/MAC → 烧录固件→ 工位 B · 测试/质检16 项测试 → QC；失败留本工位返修→ 工位 C · 装箱/发货装箱 → 标记待发货

工人动作只有三件：扫 SN → 干活 → 点一个按钮。工位屏永远只显示「当前这一台」；一张工单实时显示 37 / 100。一条「线体流水总览」显示每个工位在制/排队数，一眼看出瓶颈。

外箱不放激活码卡。外部只印 SN+二维码；激活改用「邮箱 + 扫码占有」，少一张卡、少一道工序，二手转让也更顺（见第 7 节）。

5. 包装标签 · Packaging labels

三层标签。因 Wi-Fi/蓝牙用现成（预认证）模组，须标 Contains FCC ID；简易版无 NFC。

标签位置主要内容
① 机身标贴机身LUYA / 型号 / SN+二维码 / 输入 / Contains FCC ID / UL·能效 / Made in China
② 零售彩盒标印彩盒外品名 / UPC 条码 / SN / SKU / Part15 声明 / UL·DOE VI / 责任方(进口商) / Made in China
③ 运输外箱唛头贴运输纸箱型号/SKU / ITF-14 整箱条码 / 数量 / 箱号 / 毛净重 / 搬运图标 / Made in China（走 FBA 再加 Box ID）

印外面 vs 藏箱内：外面只放 SN+二维码 + 监管标 + 零售/整箱条码；箱内不再需要任何激活码卡。

6. 种子盘制造（美国工厂 · 5 步）· Seed-tray manufacturing

消耗品在美国生产。每个托盘有唯一 ID + 二维码；仅支持 Luya 自产托盘。

流程

创建批次选种子+数量，生成每盘 ID+token→ 批量印刷标签卷整卷防水预印，提前下发→ a. 取盘→ b. 放椰糠→ c. 撒种子→ d. 包装→ e. 贴种子名+二维码

贴的码是 SN 吗？提前印还是逐盘打？

  • 人读那行是 SN（TRAY-BRO-2607-000001）+ 种子名/批次/有效期；二维码 = SN + token（token 才能验真、保证一次性、仅 Luya）。
  • 提前整卷批量预印（推荐）：种子盘高量、产线又湿又快、贴标在最后；标签即身份（托盘无内部电子件不用绑部件）。第 e 步顺序撕贴即可。
  • 「装一个打一个」只在小批量/可变数据/要逐盘扫码确认时才用——种子盘用不上。

与设备线的区别：设备要逐台扫码绑部件 SN/MAC，故必须一台台扫；种子盘没有要绑的东西，所以是「预印整卷 + 撕贴」。

7. 激活与归属（新流程）· Activation & ownership

无激活码。蓝牙近场读 SN（在机器旁）+ 邮箱，激活在连 Wi-Fi 这一步完成；第一个绑定的是拥有者；可邀成员；解绑即恢复出厂、可转让。

① 邮箱登录→ ② 蓝牙近场读 SN在机器旁，防抢注→ ③ 连 Wi-Fi（此步完成激活）→ ④ 第一个绑定=拥有者

① 谁是拥有者

第一个用「蓝牙近场读 SN + 邮箱」并连上 Wi-Fi 的账户成为 Owner——激活就在连 Wi-Fi 这一步完成。蓝牙近场要求人在机器旁：光有公开的 SN（还连续可枚举）不能激活，防止仓库/货架/物流环节被抢注，也防远程顶替。

② 家庭共享

Owner 可邀请成员（输入邮箱 + 设「可操作 / 只读」权限），多人共用同一台；别人不能自己绑，必须被邀请。

③ 出售 / 转送

Owner 在 App 点「解绑·恢复出厂」→ 释放拥有者 + 全部成员、清除 Wi-Fi/种植数据 → 设备退回「未认领」→ 下一个人扫同一 SN 即成新 Owner。

兜底（卖家忘解绑）：买家长按机身复位键即可本地清除绑定（无需原主配合），联网后云端自动释放旧绑定。possession=认领权，对低价值设备这个取舍最省客服。

操作结果
扫码 + 邮箱绑定成为 Owner，设备在线
邀请成员成员可看/操作（权限由 Owner 定）
解绑 / 恢复出厂释放所有人、抹数据、退回未认领（转让/出售）
机身物理复位无需原主即可清除绑定（二手兜底）

8. 客户使用种子盘（一次性）· Using a seed tray

扫种子盘二维码→ 校验：Luya 自产 + 未用过→ 放入机器，绑到该设备种植任务→ 按种子下发配方开始种→ 该码标记已用

二维码一次性：添加后该托盘即失效，任何账户都不能再添加。只支持 Luya 自产托盘——token 让仿冒/复用都行不通。

9. 渠道同步（订单→设备→客户）· Channel sync

销售在 Shopify/Amazon 后台运营；Luya 只读拉取订单并对接设备与客户，不回写。

客户在 Shopify/Amazon 下单→ Webhook / SP-API 通知→ Luya 只读导入→ 按邮箱匹配/新建客户→ 按归属匹配设备(SN)

这样一笔消耗品订单也能知道该补给哪一台机器；订单·设备·客户·订阅·客服全串起来。集成难度：Shopify Admin API+Webhook 中等成熟；Amazon SP-API 较重；但只需单向只读，比双向回写简单得多。

10. 安全设计要点 · Security principles

  • 三把钥匙分工：SN 公开（给人查）/ qr_token 私密（验真+防枚举+一次性）/ UUID 云端真身（不外露）。
  • 认领靠占有：蓝牙近场读 SN（在机器旁）+ 连 Wi-Fi 完成激活，而非可被偷看的箱外信息；防抢注/顶替。
  • 转让靠恢复出厂：解绑/物理复位 → 清除归属+Wi-Fi+种植数据 → 退回未认领；设备身份(UUID/SN)不变，前任数据清干净。
  • 种子盘一次性：每盘唯一 token，用过即废，仅 Luya 自产。
  • 外部不暴露密钥：箱体/标签只有 SN+二维码；激活码卡已取消。
